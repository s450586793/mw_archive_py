# v5.1.1 更新日志

> 更新日期: 2026-03-04

## 更新概述
本次更新聚焦历史归档兼容性与控制台可用性，解决 5.0 升级 5.1 后部分本地模型下载失败的问题，并优化“更新已归档页面”功能的入口与用户说明。

## 详细变更

### 新增功能
- 新增实例下载兜底接口：`GET /api/models/{model_dir}/instances/{inst_id}/download`
  - 按实例 ID 定位并返回真实 3MF 文件。
  - 下载时自动回填 `meta.json` 中实例 `fileName`，降低历史数据兼容成本。

### 优化改进
- 控制台新增标签：`归档修复`
  - 将原“更新已归档页面”功能迁移到新标签中。
  - 功能文案调整为面向普通用户的说明。
  - 按钮文案改为“一键更新历史归档”。
- `POST /api/archive/rebuild-pages` 在重建页面时增加实例文件映射扫描：
  - 自动扫描 `instances/` 并修正实例 `fileName`。
  - 返回新增统计：`fixed_instance_files`、`unresolved_instance_files`。

### Bug 修复
- 修复 5.0 历史归档在 5.1 中可能出现的实例下载链接错误问题。
  - 兼容逻辑从“依赖标题推导”调整为“优先实例文件名/名称映射”。

## 涉及文件
- `app/server.py` — 新增实例下载接口；归档重建流程加入实例映射修复逻辑。
- `app/static/js/model.js` — 实例下载链接优先按实例 ID 后端接口；文件名推断优先级修正。
- `app/templates/config.html` — 新增“归档修复”标签，调整功能说明与按钮文案，版本号更新为 `v5.1.1`。
- `README.md` — 当前版本、流程说明、API 清单与日志索引更新。
