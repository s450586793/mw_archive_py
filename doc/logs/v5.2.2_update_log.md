# v5.2.2 更新日志

> 更新日期: 2026-03-05

## 更新概述
本次为稳定性修复版本，重点解决归档时“同名配置文件被覆盖，仅保留一个”的问题，确保同名实例文件可同时保留并正确记录。

## 详细变更

### Bug 修复
- 修复归档场景下同名实例文件覆盖问题：
  - 下载保存时增加唯一化命名策略，避免后写入覆盖先写入
  - 优先使用 `name_<id>.3mf`，必要时回退 `name_<n>.3mf`，保证文件名唯一
- `meta.json` 的实例信息新增/补齐 `fileName` 字段回写，确保页面展示名与本地实际文件名解耦
- 缺失重下、按实例重下、按模型重下三条下载路径统一接入同名防覆盖逻辑，避免后续重试再次发生覆盖

## 兼容性与注意事项
- 兼容历史归档数据：旧 `meta.json` 无 `fileName` 字段的场景仍可读取
- 新归档与重下载会按新规则落盘并回写 `fileName`

## 涉及文件
- `app/archiver.py` - 新增实例文件名唯一化逻辑并回写 `instances[].fileName`
- `app/server.py` - 三条重下载路径统一接入唯一化保存策略
- `doc/bugs/BUG-20260305-001.md` - 修复记录与验证信息更新

## 验证记录
- 自测范围：同一模型包含多个同名实例文件的归档与重下载
- 自测结果：通过，生成多个不冲突文件名并全部保留
- 待验证项：线上更多模型数据的广泛回归
