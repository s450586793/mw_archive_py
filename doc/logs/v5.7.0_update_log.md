# v5.7.0 更新日志

> 更新日期: 2026-04-01

## 更新概述
本次版本围绕“归档效率提升 + 通知渠道扩展 + 跨系统路径兼容”三个方向进行增强。归档支持排队串行执行，避免手动等前一个任务结束；新增飞书通知渠道，并支持“有企业应用参数发图文、无参数自动降级文字”；模型详情页新增“模型路径”复制能力，替代跨系统直接拉起文件夹的高失败方案。

## 当前版本同步说明
- 当前版本：`v5.7.0`
- 更新说明：[`doc/logs/v5.7.0_update_log.md`](./v5.7.0_update_log.md)
- 本次重点：
  - 新增归档任务队列，支持多模型连续提交与自动串行归档；
  - 新增飞书通知渠道（Webhook + 企业应用上传图片），并支持自动文字降级；
  - 新增模型详情页“模型路径”按钮，在线页面可一键复制本地真实目录拼接路径。

## 详细变更

### 新增功能
- 新增归档队列能力：
  - `/api/archive` 改为提交任务入队；
  - 后台 `worker` 串行消费队列任务；
  - 支持查询单任务状态与队列状态：
    - `GET /api/archive/tasks/{task_id}`
    - `GET /api/archive/queue`
- 新增飞书通知渠道：
  - 支持飞书群机器人 `webhook` 推送；
  - 支持企业自建应用上传封面图后再发送图文消息；
  - 支持飞书单独测试通知按钮。
- 新增模型路径复制能力：
  - 配置页新增“本地真实目录地址”；
  - 详情页在“编辑归档”旁新增“模型路径”按钮；
  - 在线页面点击后复制 `本地真实目录 + modelDir` 到剪贴板（离线页面不显示该按钮）。

### 优化改进
- 归档重复提交优化：
  - 同一模型 URL 若已在运行或排队，自动复用已有任务，避免重复入队。
- 通知配置页优化：
  - “项目在线地址前缀”统一为通知公共配置，Telegram/飞书共用；
  - Telegram 与飞书分别提供独立“测试通知”按钮；
  - 通知页保存按钮统一放在底部，保存当前页面设置。
- 飞书发送策略优化：
  - 移除“封面上传开关”；
  - 自动根据企业应用参数是否完整决定图文或纯文字发送。

### Bug 修复
- 修复重复提交同一模型时可能产生多条重复归档任务的问题。
- 修复详情页路径辅助在离线页面展示造成误操作的体验问题（改为仅在线页面显示）。

## 兼容性与注意事项
- 归档队列当前为内存队列：
  - 服务重启后，未完成任务不会自动恢复。
- 飞书通知规则：
  - 仅配置 `webhook_url` 时发送文本；
  - 同时配置 `app_id + app_secret` 且存在封面图 URL 时发送图文；
  - 上传失败会自动降级为文本通知。
- 路径复制功能：
  - 仅负责“复制路径”，不直接调用系统打开文件夹；
  - 复制是否成功受浏览器剪贴板权限影响。

## 涉及文件
- `app/server.py` - 归档队列调度、去重入队、任务查询接口、飞书配置接口、路径复制配置接口
- `app/templates/config.html` - 通知页改版（公共地址、分渠道测试、统一保存）、路径兼容配置入口
- `app/templates/model.html` - 详情页新增“模型路径”按钮
- `app/static/js/model.js` - 模型路径拼接与复制逻辑（仅在线页面展示）
- `app/static/css/model.css` - 详情页按钮同排与配色优化
- `app/feishu_push.py` - 新增飞书通知服务（文本/图文自动切换）
- `app/config/config.json` - 新增/精简相关配置结构
