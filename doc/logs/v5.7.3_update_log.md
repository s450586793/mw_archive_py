# v5.7.3 更新日志

> 更新日期: 2026-04-08

## 更新概述
本次版本是一次补丁修复发布，重点解决两个稳定性问题：一是摘要图片下载失败会拖垮整个归档任务，二是模型目录名包含 `#` 时图库封面、头像等图片资源无法正常显示。现在归档遇到正文图片 `403` 时会继续完成，图库也能正确处理包含特殊字符的目录名。

## 当前版本同步说明
- 当前版本：`v5.7.3`
- 更新说明：[`doc/logs/v5.7.3_update_log.md`](./v5.7.3_update_log.md)
- 本次重点：
  - 修复摘要图片 `403` 导致整个模型归档失败的问题；
  - 摘要图片下载失败时改为保留原始远程链接，整体归档不中断；
  - 修复模型目录名包含 `#` 时图库封面、头像、灯箱图片加载失败的问题；
  - 新增对应回归测试与 Bug 记录文档。

## 详细变更

### 新增功能
- 无

### 优化改进
- 优化摘要内容容错策略：
  - 成功下载的摘要图片继续本地化；
  - 下载失败的摘要图片保留原始远程地址；
  - 后续摘要内容与整体模型归档继续执行。
- 优化图库文件资源地址构建策略：
  - 统一对 `model_dir` 做 URL 编码；
  - 相对路径按段编码并保留 `/`；
  - 降低目录名包含特殊字符时的资源加载失败概率。

### Bug 修复
- 修复 MakerWorld 部分摘要图片返回 `403 Forbidden` 时，整个归档任务被直接终止的问题。
- 修复模型目录名包含 `#` 时，首页图库封面图、作者头像和灯箱预览无法正常显示的问题。

## 兼容性与注意事项
- 本次为兼容性补丁版本，不涉及配置结构变更。
- 若摘要图片源链接本身已失效，离线页中该图片会保留远程地址，显示效果取决于源站是否可访问。
- 图库资源路径改为统一编码后，旧数据无需迁移即可直接生效。
