## 1. 用户需求记录（原始需求 + 初步思路）
- 需求描述：
  - 希望在模型详情页增加“快速打开本地文件夹”能力。
  - 在配置页增加服务器 IP 和真实路径配置项。
  - 通过拼接模型项目文件夹名，定位到对应模型目录并打开。
  - 需要分析 Windows 和 macOS 的打开方式。
- 用户初步实现思路：
  - 在配置页保存服务器 IP 与基础真实路径。
  - 详情页读取当前模型 `modelDir`，与基础路径拼接出目标目录。
  - 点击按钮后直接打开系统文件管理器到该目录。
- 目标效果：
  - 在详情页可一键跳转到当前模型的本地目录，减少手动检索。
- 约束/注意事项：
  - 不能允许任意路径拼接，需限制在配置根目录下。
  - 需兼容中文、空格、特殊符号目录名。
  - 需考虑浏览器对 `file://`、`smb://`、本地程序唤起的限制。

## 2. AI 分析（难度 / 思路 / 改动范围）
- 实现难度：中
- 总体实现思路：
  - 配置层新增“目录打开”配置，例如：
    - `folder_open.server_ip`
    - `folder_open.real_root_path`
    - 可选补充：`folder_open.client_path_windows`、`folder_open.client_path_macos`
  - 详情页基于当前 URL 中的 `modelDir` 直接取得当前模型目录名；该值已在前端 `app/static/js/model.js` 中解析，可复用。
  - 后端返回详情数据时补充“目录打开信息”，或由前端直接将 `modelDir` 与配置拼接成目标路径。
  - 详情页增加“打开本地文件夹”按钮，点击后按平台生成目标地址。
- 当前代码基础：
  - `app/config/config.json` 已有统一配置文件，适合新增目录打开配置。
  - `app/templates/config.html` 已有配置页表单与保存逻辑，适合增加输入项。
  - `app/templates/model.html` 与 `app/static/js/model.js` 已有详情页操作区，适合增加按钮与事件。
  - `app/server.py` 已有 `resolve_model_dir()`、`download_dir` 等目录处理逻辑，适合补路径校验与配置持久化。
- Windows 打开方案分析：
  - 方案 A：拼接 UNC / `file://` 路径，例如 `file://192.168.1.10/share/models/LocalModel_xxx`
  - 优点：实现最轻，不需要后端执行系统命令。
  - 风险：
    - 浏览器可能拦截 `file://` 跳转。
    - 如果配置的是服务器“真实路径”（如 `D:\archive\data`），客户端无法直接访问，必须是共享路径或本地映射盘路径。
  - 更稳方案：
    - 配置 Windows 客户端可访问路径（如 `\\192.168.1.10\archive\data`），前端直接打开该路径。
    - 或由运行在 Windows 的后端提供 `/api/open-folder`，服务端调用 `explorer.exe` 打开目录。
- macOS 打开方案分析：
  - 如果目录已通过 SMB 挂载到 Finder，最稳的是拼接本机挂载路径，例如 `/Volumes/archive/data/LocalModel_xxx`，再使用 `file://`。
  - 如果希望通过服务器 IP 直连，可尝试 `smb://192.168.1.10/archive/data/LocalModel_xxx`，由 Finder 处理。
  - 风险：
    - macOS 通常不能直接使用服务器的“真实磁盘路径”如 `/srv/archive/data`，除非该路径正好是共享根并已映射。
    - 因此仅配置“服务器 IP + 真实路径”对 macOS 不一定足够，通常还需要“客户端挂载路径”或“共享名”。
- 推荐实现结论：
  - 若追求低成本前端方案，建议把配置从“真实路径”升级为“客户端可访问路径”：
    - Windows：`\\server\share\archive`
    - macOS：`/Volumes/archive` 或 `smb://server/share`
  - 若必须保留“服务器 IP + 真实路径”这两个字段，建议再加一层路径映射规则：
    - `server_real_root`：服务端真实路径，仅用于展示和校验
    - `client_open_root_windows` / `client_open_root_macos`：客户端实际打开路径
  - 若希望用户点击后 100% 可靠打开，最佳方案是做本机辅助能力：
    - 后端与浏览器在同一台机器时，由后端直接调用系统命令打开目录
    - Windows：`explorer.exe "<path>"`
    - macOS：`open "<path>"`
- 预计改动模块：
  - `app/server.py` 配置结构、保存接口、路径拼接与安全校验
  - `app/templates/config.html` 新增目录打开配置项
  - `app/templates/model.html` 增加“打开本地文件夹”按钮
  - `app/static/js/model.js` 生成目标目录并处理打开动作
  - `app/static/css/model.css` 按钮样式
- 主要风险点：
  - 浏览器安全策略可能拦截 `file://` 或 `smb://`
  - macOS 与 Windows 的路径格式不一致，单一“真实路径”难同时兼容
  - 中文与特殊字符需做 URL 编码，否则可能再次出现路径跳转异常
  - 若开放后端执行打开目录，需严格限制目标路径必须落在配置根目录下
- 验证方案：
  - 配置 Windows 路径后，在详情页点击打开，验证可定位到当前 `modelDir`
  - 配置 macOS 挂载路径或 `smb://` 路径后验证 Finder 是否正常拉起
  - 使用含空格、中文、括号、百分号的目录做跳转验证
  - 构造非法目录名与越权路径，确认被拦截
