## 1. 用户需求记录（原始需求 + 初步思路）
- 需求描述：
  - 新增飞书推送渠道。
  - 发送链路为：
    - 企业自建应用接口上传封面图；
    - 飞书群机器人 webhook 推送消息。
  - 若未配置企业应用参数，飞书仍可发送纯文字通知。
  - 页面需支持配置并持久化到 `config.json`。
- 用户初步实现思路：
  - 沿用现有通知配置页，新增飞书配置区域和测试能力。
- 目标效果：
  - 飞书可作为独立通知渠道使用，并支持图文/文字自动切换。
  - 配置页可保存飞书参数，并可单独测试飞书通知。
- 约束/注意事项：
  - 图文能力依赖企业应用参数；参数缺失时需自动降级文本，不应报错中断。

## 2. AI 分析（难度 / 思路 / 改动范围）
- 实现难度：中
- 总体实现思路：

- 验证方案：
  - 仅配置 webhook，验证文本通知可发。
  - 配置 `app_id + app_secret` 后验证图文通知可发。

## 3. AI 实现记录（实际落地）
- 实际改动文件：

  - 通知页改版：

    - 保存按钮统一放到通知页底部；
