## 1. 用户需求记录（原始需求 + 初步思路）
- 需求描述：
  - 在 MakerWorld 模型详情页打开时，油猴脚本或 Edge 插件自动调用本地后端接口，按模型 ID 查询该模型是否已归档。
  - 在页面标题旁显示归档状态标记（已归档 / 未归档）及归档时间。
  - 该能力需要可配置开关，关闭时不请求接口、不显示标记。
- 用户初步实现思路：
  - 后端新增一个“最小数据处理”的查询接口，模型 ID 可从本地索引文件快速查找。
  - 前端（油猴 / 扩展 content script）仅负责提取模型 ID、请求接口并渲染标记。
- 目标效果：
  - 浏览模型页时能即时判断是否已经归档过此模型。
  - 基于“归档时间 + 页面信息”辅助判断模型是否可能有更新。
- 约束/注意事项：
  - 仅在模型详情页生效。
  - 开关关闭时不产生额外网络请求。
  - 接口优先走索引查找，避免扫描全盘目录造成延迟。
  - 更新判断暂不自动计算，仅展示归档日期供用户自行判断。
  - 状态样式：已归档（绿）/ 未归档（灰）。
  - 默认关闭该功能开关（需求原始约束）；首版联调阶段按用户要求“油猴默认开启”验证挂载位。
  - 索引未命中时直接显示“未归档”，不做慢速目录扫描。

## 2. AI 分析（难度 / 思路 / 改动范围）
- 实现难度：中
- 当前代码现状：
  - 油猴脚本：`plugin/tampermonkey/mw_quick_archive.user.js`，已有后端地址配置与归档动作。
  - Edge/Chrome 扩展：`plugin/chrome_extension/mw_quick_archive_ext`，`background.js + content.js + options.js` 结构已可调用本地后端。
  - 后端已维护图库索引 `app/config/gallery_index.json`，单条索引项包含 `id`（模型ID）和 `collectedAt`（归档时间），适合做最小查询来源。
- 总体实现思路：
  - 后端新增接口（建议）：
    - `GET /api/archive/status/{model_id}`
    - 返回最小字段：`exists`、`archived_at`、`model_dir`、`title`、`source`。
  - 查询逻辑：
    - 优先读取 `gallery_index.json` 的 `items`，按 `id == model_id` 匹配。
    - 若索引缺失或未命中，可选是否回退目录扫描（建议默认不回退，保持“最小处理”）。
  - 前端脚本逻辑：
    - 从 URL 提取模型 ID（`/zh/models/{id}`）。
    - 读取“归档状态标记开关”。
    - 开启时请求接口，在页面标题区域插入状态徽标。
    - 标记内容示例：`本地已归档 · 2026-04-01 14:20` / `本地未归档`。
  - 更新判断策略（建议分阶段）：
    - 第一阶段：仅显示“是否归档 + 归档时间”。
    - 第二阶段（可选）：如后续需要再补“可能有更新”自动判断。
  - 标记插入位置策略：
    - 需要依赖页面标题 DOM 节点进行定位。
    - 采用“标题主选择器 + 多级候选选择器 + MutationObserver 重试”的兼容方式，降低页面结构变动带来的失效概率。
  - 已确认候选挂载点（用户提供）：
    - 候选 A（优先）：标题节点后插入
      - 容器：`div.MuiBox-root.mw-css-1a5vpwx`
      - 标题：`h1.title-for-share.translated-text`
      - 挂载策略：在 `h1` 后追加状态徽标，视觉上最接近标题语义。
    - 候选 B（回退）：已关注按钮区域并排插入
      - 容器：`div.mw-css-wn1ugj`
      - 参考按钮：`button[designid]`（文本常见为“已关注”）
      - 挂载策略：在按钮后追加徽标，作为操作区补充状态信息。
    - 失败回退：若 A/B 都未命中，则本次不渲染标记（避免误插入）。
- 预计改动模块：
  - `app/server.py`
  - `plugin/tampermonkey/mw_quick_archive.user.js`
  - `plugin/chrome_extension/mw_quick_archive_ext/content.js`
  - `plugin/chrome_extension/mw_quick_archive_ext/background.js`
  - `plugin/chrome_extension/mw_quick_archive_ext/options.js`
  - `plugin/chrome_extension/mw_quick_archive_ext/options.html`
- 主要风险点：
  - MakerWorld 页面结构变动可能影响“标题旁插入位置”稳定性。
  - 模型 ID 与本地索引 `id` 字段需保证同一口径（数字或字符串）。
  - 扩展与油猴都加请求时需控制频率，避免重复查询。
  - `mw-css-*` 类名可能由构建产物生成，存在变更风险，需搭配语义类名/标签组合选择器与重试机制。
- 验证方案：
  - 有归档模型：显示“已归档 + 时间”。
  - 未归档模型：显示“未归档”。
  - 关闭开关：无请求、无标记。
  - 切换 `makerworld.com` / `makerworld.com.cn` 均可识别。
