## 1. 用户需求记录（原始需求 + 初步思路）
- 需求描述：
  - 当前 Telegram 和飞书的推送标题、正文分别在各自模块中单独维护。
  - 希望先完成第一步：统一标题和内容模板，由各通知渠道复用。
  - 同时为后续支持“用户自定义修改模板”预留扩展点。
- 用户初步实现思路：
  - 抽一个公共模板层。
  - 所有通知直接调用公共模板。
- 目标效果：
  - 成功通知、告警通知的标题和正文统一维护。
  - TG / 飞书文案输出保持一致。
  - 后续自定义模板时不需要再改各渠道发送逻辑。
- 约束/注意事项：
  - 本次只做第一阶段，不开放配置界面或用户可编辑模板。
  - 尽量不改变已有推送字段和发送链路。

## 2. AI 分析（难度 / 思路 / 改动范围）
- 实现难度：低
- 总体实现思路：
  - 新增独立通知模板模块，负责产出统一的标题、正文、完整文本。
  - `tg_push.py` 和 `feishu_push.py` 仅负责渠道发送，不再各自拼接通知文案。
  - 飞书图文卡片标题也改为复用统一模板标题，避免只统一正文。
- 预计改动模块：
  - `app/notification_templates.py`
  - `app/tg_push.py`
  - `app/feishu_push.py`
  - `tests/notify.duration.test.py`
- 主要风险点：
  - 模板抽离后字段缺失导致推送内容为空或格式异常。
  - 飞书图文标题与正文如果仍分开维护，会形成半统一状态。
- 验证方案：
  - 校验模板模块语法。
  - 校验 TG / 飞书成功通知文本一致。
  - 校验归档耗时、缺失 3MF 等字段仍能正常出现在推送内容中。
