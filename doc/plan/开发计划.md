# MakerWorld Archive 开发计划

更新时间: 2026-03-04

## 一、目标
- 提升归档稳定性: 降低 cookie 限制、失败率和人工干预成本。
- 提升可用性: 增加通知、批量导入、在线编辑等核心能力。
- 提升可运营性: 提供可观测、可恢复、可扩展的任务体系。

## 二、当前需求拆解

### 1. Telegram 推送与交互
目标:
- 归档成功后推送结果。
- cookie 失效、限流、任务失败时告警。
- 支持通过 Telegram 发送模型地址触发自动归档。

设计建议:
- 配置项: `bot_token`、`chat_id`、`enable_push`、`enable_command`。
- 消息模板:
  - 成功: 模型第一张图片 + 模型标题 + 链接 + 归档时间。
  - 失败: 错误类型 + 重试建议。
  - cookie 告警: 失效 cookie 标识 + 影响范围。
- 命令建议:
  - 直接发送 https://makerworld.com.cn/zh/models/* 地址 ,归档模型
  - `/cookies`: 查看 cookie 池健康度摘要。
  - `/count`: 查询模型总数

验收标准:
- 可在页面配置 Telegram 并保存。
- 成功/失败/cookie 异常均有对应推送。
- 机器人收到链接后可入队并执行归档。

### 2. 本地模型批量导入
目标:
- 批量扫描本地模型文件并导入模型库。

关键问题:
- 并发策略:
  - 方案 A: 单进程队列（默认，稳定优先）。
  - 方案 B: 有上限并发（可配置，如 2-4 并发）。
- 多配置识别:
  - 建议用 `模型主键 + 配置指纹` 两层结构。
  - 主键可由模型名称标准化 + 作者 + 来源路径生成。
  - 配置层用参数哈希（喷嘴、层高、材料等）区分。

验收标准:
- 支持目录批量导入。
- 可正确聚合同模型多配置，不重复建主模型。
- 支持导入报告（成功/跳过/失败）。

### 3. 多 Cookie 轮询
目标:
- 降低单 cookie 被限流或失效导致的中断。

设计建议:
- cookie 池状态机: `active` / `cooldown` / `invalid`。
- 调度策略:
  - 默认轮询 + 失败退避。
  - 命中限流后自动冷却一定时间。
- 健康检查:
  - 周期性验证 cookie 可用性。
  - 后台统计成功率、失败率、最近错误。

验收标准:
- 支持新增多个 cookie。
- 可自动切换 cookie 执行任务。
- 单 cookie 失效不会阻塞整体任务。

### 4. 在线编辑已归档模型
目标:
- 在本地模型页面直接编辑归档内容并保存。

建议范围:
- 可编辑字段: 标题、标签、说明、分类、版本备注。
- 编辑保护:
  - 字段级校验（长度、必填、非法字符）。
  - 变更审计（谁在何时改了什么）。
- 回滚能力:
  - 保留最近 N 次历史版本，支持回滚。

验收标准:
- 可在 UI 中编辑并保存。
- 保存后前台展示与检索结果同步更新。
- 错误输入有明确提示。

## 三、补充规划建议
- 任务中心: 统一查看任务队列、重试、取消、失败原因。
- 幂等与去重: 同一 URL 重复提交自动合并任务。
- 异常恢复: 服务重启后自动恢复未完成任务。
- 权限与安全:
  - Telegram 命令白名单（仅允许指定 chat/user）。
  - 敏感配置（token/cookie）加密存储。
- 观测能力:
  - 关键指标（成功率、平均耗时、失败分布）。
  - 结构化日志 + 错误码体系。
- 数据备份:
  - 模型库定期导出与恢复脚本。

## 四、建议里程碑

### M1（1-2 周）稳定性基础
- 多 cookie 池 + 轮询调度 + 健康检查。
- 任务队列基础（入队、执行、失败重试）。
- 基础告警（cookie 失效、任务失败）。

### M2（1-2 周）通知与自动化
- Telegram 推送配置页。
- 成功/失败/告警消息模板。
- Telegram `/archive` 命令触发入队。

### M3（2 周）批量导入
- 本地目录扫描与批量入队。
- 单进程队列先落地，再开放可配置并发。
- 同模型多配置聚合规则与导入报告。

### M4（1-2 周）内容维护
- 在线编辑归档模型。
- 历史版本与回滚。
- 编辑审计日志。

## 五、优先级建议
- P0: 多 cookie + 任务队列 + 基础告警（先保稳定）。
- P1: Telegram 推送与命令归档（提高自动化）。
- P2: 批量导入 + 多配置聚合（提升效率）。
- P3: 在线编辑与版本回滚（完善运营能力）。

## 六、待确认决策
- 批量导入默认并发数（建议默认 1，可配置上限）。
- 模型主键规则是否需要人工干预入口（合并/拆分）。
- Telegram 命令权限范围（单 chat 还是多 chat）。
- 历史版本保留策略（按数量或按时间）。
